## 🚀 Why we Used MERN Stack for BloodCampus

The **BloodCampus** project is built using the MERN Stack to ensure scalability, performance, and a smooth full-stack architecture.

| Technology | Purpose |
|------------|----------|
| <img src="https://img.shields.io/badge/MongoDB-47A248?style=flat-square&logo=mongodb&logoColor=white"/> | Stores donor and user data efficiently with a flexible NoSQL database structure. |
| <img src="https://img.shields.io/badge/Express.js-000000?style=flat-square&logo=express&logoColor=white"/> | Handles backend logic, REST APIs, and server-side routing. |
| <img src="https://img.shields.io/badge/React-20232A?style=flat-square&logo=react&logoColor=61DAFB"/>  | Builds a fast, responsive, and dynamic user interface. |
| <img src="https://img.shields.io/badge/Node.js-339933?style=flat-square&logo=nodedotjs&logoColor=white"/> | Runs server-side code and ensures high performance. |

✅ MERN enables real-time donor management, secure authentication, and scalable system design for BloodCampus.

✅ The MERN stack enables BloodCampus to deliver real-time donor management, secure authentication, and a scalable full-stack architecture.




## 🔐 Authentication & Authorization System

BloodCampus ensures secure user access and protected resources using modern authentication and authorization methods.

| Technology | Purpose |
|------------|----------|
| <img src="https://img.shields.io/badge/Firebase-FFCA28?style=flat-square&logo=firebase&logoColor=black"/>| Handles secure user registration, login, logout, and identity verification. |
| <img src="https://img.shields.io/badge/JWT-000000?style=flat-square&logo=jsonwebtokens&logoColor=white"/>| Provides token-based authentication for secure API communication and protected routes. JWT (JSON Web Token) |
| 🛡️ **Role-Based Authorization** | Controls system access based on user roles (Donor, User) to protect sensitive features. |

✅ Ensures secure login, protected routes, role-based access control, and safe data handling.
